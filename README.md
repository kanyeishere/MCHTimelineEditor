# FF14 机工士 PVE 技能时间轴模拟器

这是一个无外部运行时依赖的静态网页应用。入口文件是 `index.html`，交互逻辑在 `app.js`，样式在 `styles.css`；`npm run build` 会把这些文件复制到 `dist/`，供 GitHub Pages 发布。

## 本地预览

```bash
npm start
```

然后打开 <http://localhost:5173>。


## 用户简易使用说明

### 1. 开始排轴

1. 打开网页后，上方是机工士 PVE 技能图标面板。
2. 可以把技能图标拖到下方时间轴，也可以先点击时间轴中的目标格子，再点击技能图标快速添加。
3. GCD 战技放在 `GCD` 行；能力技放在 `能力技 1 / 2 / 3` 行。短 GCD（如烈焰弹）后只允许插入 2 个能力技。
4. 已经放在时间轴上的技能可以继续拖动到其他格子；点击已放置技能可以移除。

### 2. 资源、Buff 与 CD 提醒

- `Heat / Battery` 行会显示热量和电量变化。
- 顶部的 `初始热量`、`初始电量` 可以用来模拟 P2、P3 等后续阶段开场资源。
- `主要CD` 行会提示钻头、空气锚、回转飞锯、枪管加热、野火、爆发药水等关键技能何时转好；点击提示图标可以尝试把该技能添加到对应列。
- `Buff` 行会显示爆发药、野火、超荷预备、掘地飞轮预备、全金属爆发预备等状态；其中部分预备 Buff 图标可以点击，直接添加对应技能。
- `Double Check / Checkmate / Reassemble` 行会显示双将、将死、整备的层数恢复情况。

### 3. 机器人与爆发窗口

- 放置 `后式自走人偶` 时，会消耗当前全部电量，并在机器人行显示本次消耗的电量。
- 机器人从出现到消失期间，机器人行会以蓝色背景标记。
- 使用爆发药水后，药效 30 秒内的本体技能与机器人技能会高亮，方便检查爆发覆盖。

### 4. 快捷添加

- `+ 基础连`：在最后一个已放置 GCD 后面追加 `热分裂弹 → 热独头弹 → 热狙击弹`。
- `+ 过热连`：在最后一个已放置 GCD 下方放入 `超荷`，并追加 5 个 `烈焰弹`。如果能力技槽不足或资源/预备条件不满足，会提示无法添加。
- `+ 转场空白`：可以把当前 GCD 后面插入一段 Boss 上天/转场空白，并自定义名称（如 `P2转场`）和时长。也可以先选中已有空白区域，再点击该按钮，把原本的空白压缩成一个转场列。

### 5. 时间轴操作

- 鼠标滚轮在时间轴区域内会横向滚动。
- 按住 `Ctrl` 再滚动鼠标滚轮，可以横向缩放时间轴。
- 时间轴会自动延伸到最后一个已放置 GCD 之后至少 240 秒，因此可以继续向后排更长的轴。

### 6. 保存与分享

- 点击 `导出` 会下载当前时间轴 JSON 文件。
- 点击 `导入` 可以恢复之前导出的 JSON。
- 导入内容会保留技能安排、初始热量/电量、转场空白等信息。
- 如果导入文件中包含当前规则下不合法的技能（例如缺少预备 Buff、资源不足），应用会自动移除并提示数量。

## 部署到 GitHub Pages（推荐：GitHub Actions）

仓库已经包含 `.github/workflows/pages.yml`，每次推送到 `main` 分支时会自动构建并发布 `dist/` 到 GitHub Pages。

第一次启用时需要在 GitHub 仓库页面做一次设置：

1. 打开仓库的 **Settings**。
2. 左侧进入 **Pages**。
3. 在 **Build and deployment** 里，把 **Source** 选择为 **GitHub Actions**。
4. 确认默认分支名是 `main`；如果你的默认分支不是 `main`，请同步修改 `.github/workflows/pages.yml` 中的 `branches: [main]`。
5. 推送代码到 GitHub：

```bash
git push origin main
```

6. 到仓库的 **Actions** 页面等待 `Deploy static site to GitHub Pages` 工作流完成。
7. 部署成功后，访问地址通常是：

```text
https://<你的用户名>.github.io/<仓库名>/
```

例如仓库名是 `MCHTimelineEditor`，地址通常类似：

```text
https://<你的用户名>.github.io/MCHTimelineEditor/
```

## 备用部署方式：直接从分支发布

因为本项目根目录本身就是静态站点，你也可以不用 Actions：

1. 打开 **Settings → Pages**。
2. **Source** 选择 **Deploy from a branch**。
3. **Branch** 选择 `main`，目录选择 `/root`。
4. 保存后等待 GitHub Pages 发布。

如果使用这种方式，GitHub 会直接发布根目录的 `index.html`、`app.js`、`styles.css`。
